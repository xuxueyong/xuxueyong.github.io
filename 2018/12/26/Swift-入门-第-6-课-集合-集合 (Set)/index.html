Swift 入门 第 6 课 集合-集合（Set)

集合（Set）

数据要求不按顺序， 且不能有重复的元素出现， 用Set。

Swift 的 Set 可以被桥接到 Objective-C 的 NSSet

集合类型的哈希值

一个类型为了存储在集合中，该类型必须是可哈希化的–也就是说，该类型必须提供一个方法来计算它的哈希值。一个哈希值是 Int 类型的，相等的对象哈希值必须相同，比如 a==b,因此必须 a.hashValue == b.hashValue。

Swift 的所有基本类型（比如 String,Int,Double 和 Bool）默认都是可哈希化的，可以作为集合的值的类型或者字典的键的类型。没有关联值的枚举成员值（在枚举有讲述）默认也是可哈希化的。

注意:
你可以使用你自定义的类型作为集合的值的类型或者是字典的键的类型，但你需要使你的自定义类型符合

Swift 标准库中的 Hashable 协议。符合 Hashable 协议的类型需要提供一个类型为 Int 的可读属>性 hashValue。由类型的 hashValue 属性返回的值不需要在同一程序的不同执行周期或者不同程序之间保持相同。

因为 Hashable 协议符合 Equatable 协议，所以遵循该协议的类型也必须提供一个“是否相等”运算符（==）的实现。这个 Equatable 协议要求任何符合 == 实现的实例间都是一种相等的关系。也就是说，对于 a,b,c 三个值来说，== 的实现必须满足下面三种情况：

a == a(自反性)
a == b 意味着 b == a(对称性)
a == b && b == c 意味着 a == c(传递性)

访问和修改一个集合

你可以通过 Set 的属性和方法来访问和修改一个 Set。

isEmpty 判断集合是否为空

insert(_:) 方法来添加一个新元素

remove 方法可以删除对应元素， 如果删除的元素在集合中不存在， 则返回 nil

contains 方法用来判断集合中 是否有指定的值存在， 数组也可以使用
